# 依赖注入系统实现计划

## 目标

为 RPCServer 添加灵活的依赖注入系统，允许开发者自定义需要注入的依赖，而不仅限于 IOWrite。

## 当前状态

* 仅支持 `IOWrite` 的硬编码注入（application.py#L238-240）

* 注入逻辑写死在 `__execute_method` 方法中

* 没有依赖注册和管理机制

## 设计方案

### 1. 创建依赖注入容器类 (DependencyContainer)

**文件**: `src/okstdio/server/dependencies.py` (新建)

**功能**:

* 存储和管理所有可注入的依赖

* 提供依赖注册接口

* 支持按类型查找依赖

* 支持依赖的生命周期管理（单例/每次请求创建）

**核心方法**:
- `register(key: Type | str, factory: Callable, singleton: bool = True)`: 注册依赖（支持类型或字符串作为键）
- `get(key: Type | str) -> Any`: 获取依赖实例
- `resolve_parameter(param_type: Type) -> Any | None`: 根据类型解析依赖
- `has(key: Type | str) -> bool`: 检查依赖是否已注册

### 2. 修改 RPCServer 类

**文件**: `src/okstdio/server/application.py`

**改动**:

* 在 `__init__` 中初始化 `DependencyContainer`

* 提供 `register_dependency` 方法供开发者注册自定义依赖

* 修改 `__execute_method` 方法，使用依赖容器而不是硬编码判断

* 保留 `IOWrite` 作为内置依赖自动注册

**新增方法**:
```python
def register_dependency(self, key: Type | str, factory: Callable, singleton: bool = True) -> None:
    """注册依赖
    
    Args:
        key: 依赖的标识符，可以是类型或字符串
        factory: 依赖工厂函数
        singleton: 是否单例，默认 True
    """

def get_dependency(self, key: Type | str) -> Any:
    """获取依赖实例
    
    Args:
        key: 依赖的标识符
        
    Returns:
        依赖实例
    """

def has_dependency(self, key: Type | str) -> bool:
    """检查依赖是否已注册
    
    Args:
        key: 依赖的标识符
        
    Returns:
        是否已注册
    """
```

**修改逻辑**:

```python
# 原代码（硬编码）
elif ann is IOWrite:
    bound_args[name] = IOWrite(self)

# 新代码（使用依赖容器）
elif (dep := self._dependency_container.resolve_parameter(ann)) is not None:
    bound_args[name] = dep
```

## 使用示例

### 方式一：服务器创建时注册（已知依赖配置）
```python
from okstdio.server import RPCServer
import uiautomator2 as u2

app = RPCServer("app")

# 如果已知设备地址，可以直接注册
app.register_dependency(
    u2.Device, 
    lambda: u2.connect("192.168.1.100:5555"),
    singleton=True
)
```

### 方式二：延迟注册（推荐 - 运行时动态注册）
```python
from okstdio.server import RPCServer
import uiautomator2 as u2

app = RPCServer("app")

# 服务器启动时不知道设备信息，延迟到方法中注册
@app.add_method()
def init_device(device_ip: str, device_port: int) -> dict:
    # 在方法中动态注册依赖
    device = u2.connect(f"{device_ip}:{device_port}")
    app.register_dependency(u2.Device, lambda: device, singleton=True)
    return {"status": "device registered"}

# 注册后，其他方法可以直接使用 device 依赖
@app.add_method()
def click_device(device: u2.Device) -> dict:
    device.click(0.5, 0.5)
    return {"status": "ok"}
```

### 方式三：使用依赖工厂（高级用法）
```python
# 注册一个工厂函数，根据参数创建不同的设备
app.register_dependency(
    "device_factory",  # 使用字符串标识符
    lambda device_ip: u2.connect(device_ip),
    singleton=False
)

@app.add_method()
def use_device(device_ip: str) -> dict:
    factory = app.get_dependency("device_factory")
    device = factory(device_ip)
    device.click(0.5, 0.5)
    return {"status": "ok"}
```

### 高级用法（非单例 - 每次请求创建新实例）
```python
# 某些场景需要每次请求都创建新实例（如数据库会话）
app.register_dependency(
    Session, 
    lambda: Session(), 
    singleton=False  # 每次请求都创建新实例
)

@app.add_method()
def process(session: Session):
    # 每次调用都会创建新的 Session
    pass
```

### 内置 IOWrite 依赖

```python
# IOWrite 自动注册，无需手动注册
@app.add_method()
async def long_task(io_write: IOWrite):
    await io_write.write({"result": "progress"})
```

## 技术要点

### 依赖键类型
- **类型键**: 使用 Python 类型作为键（如 `u2.Device`），用于自动参数注入
- **字符串键**: 使用字符串作为键（如 `"device_factory"`），用于工厂函数或复杂场景

### 依赖解析策略
1. **按类型匹配**: 检查参数类型是否在依赖容器中注册
2. **支持子类**: 使用 `isinstance` 或类型层次结构检查
3. **默认值处理**: 如果依赖不存在且有默认值，使用默认值
4. **运行时注册**: 支持在方法执行过程中动态注册依赖

### 生命周期管理
- **单例 (singleton=True)**: 第一次请求时创建实例，后续所有请求都复用这个实例
  - 适用于：设备连接、数据库连接池、配置对象等
  - 优点：避免重复创建开销，保持状态
- **非单例 (singleton=False)**: 每次请求都创建新实例
  - 适用于：临时会话、请求上下文等
  - 优点：隔离性好，避免状态污染

### 线程安全
- 依赖容器应该是线程安全的
- 单例依赖的创建应该使用锁机制避免重复创建

### 错误处理

* 如果依赖无法创建，抛出清晰的错误信息

* 提供依赖缺失的调试信息
