# 服务器方法树功能实现计划

## 问题分析

当前 RPCServer 已经通过 `AppDoc` 类提供了 `docs_json()` 方法，可以生成完整的方法树文档。但是这个功能存在以下问题：

1. **没有暴露为 RPC 方法**：客户端无法通过 RPC 调用获取方法树
2. **文档生成与运行时分离**：`docs_json()` 主要用于生成静态文档，而不是运行时查询
3. **缺少标准化的方法树接口**：需要一个标准的 RPC 方法让客户端查询服务器能力

## 实现目标

为 RPCServer 添加一个默认的内置方法（如 `__system__` 或 `system.listMethods`），让 RPCClient 可以通过 RPC 调用获取服务器的方法树。

## 实现方案

### 方案一：系统方法（推荐）

在 RPCServer 中内置一个系统方法，例如 `__system__` 或 `system.listMethods`，该方法返回服务器的方法树结构。

**优点**：
- 符合 JSON-RPC 规范的系统方法命名约定
- 客户端可以通过标准接口查询服务器能力
- 不影响用户自定义方法

**实现步骤**：

1. **在 RPCServer 中添加系统方法**
   - 方法名：`__system__`（或 `system.listMethods`）
   - 功能：返回服务器的完整方法树
   - 参数：可选的 `detail` 参数，控制返回信息的详细程度
   - 返回值：方法树字典

2. **复用现有的文档生成逻辑**
   - 将 `AppDoc.docs_json()` 的核心逻辑提取出来
   - 系统方法直接调用该逻辑生成方法树
   - 避免代码重复

3. **在 RPCRouter 中注册系统方法**
   - 系统方法在路由器初始化时自动注册
   - 使用特殊的命名空间（如 `__system__`）避免与用户方法冲突
   - 系统方法不参与路由分发（特殊处理）

4. **可选：增强方法树信息**
   - 添加方法调用次数统计
   - 添加方法注册/最后调用时间
   - 添加依赖注入信息

### 方案二：独立的系统路由器

创建一个专门的系统路由器，包含多个系统方法：
- `system.listMethods`：列出所有方法
- `system.methodInfo`：获取单个方法详情
- `system.describe`：获取服务器描述信息

**优点**：
- 功能更加丰富
- 扩展性更好

**缺点**：
- 实现复杂度较高
- 可能过度设计

## 详细实现（采用方案一）

### 1. 修改 `RPCServer.__init__()`

在初始化时自动注册系统方法：

```python
def __init__(self, server_name: str = "app", label: str = "", version: str = "v0.1.0"):
    self.server_name = server_name
    self.version = version

    StdioStream.__init__(self)
    RPCRouter.__init__(self, server_name, label)
    
    # 初始化依赖注入容器
    self._dependency_container = DependencyContainer()
    
    # 自动注册内置依赖 IOWrite
    self._dependency_container.register(IOWrite, lambda: IOWrite(self), singleton=True)
    
    # 注册系统方法
    self._register_system_methods()

def _register_system_methods(self):
    """注册系统方法"""
    # 使用特殊的前缀，确保不会与用户方法冲突
    self.methods["__system__"] = (self.__system_info__, "系统信息")

async def __system_info__(self, detail: bool = False) -> dict:
    """获取服务器系统信息
    
    Args:
        detail: 是否返回详细信息，默认 False
    
    Returns:
        包含服务器方法树的字典
    """
    # 复用 docs_json() 的逻辑
    return self.docs_json()
```

### 2. 修改 `RPCServer.handle_request()`

在请求分发时，特殊处理系统方法：

```python
async def handle_request(self, request_string: str) -> JSONRPCResponse:
    # ... 现有代码 ...
    
    async def dispatch(current: RPCRouter, parts: list[str]) -> JSONRPCResponse:

        head, *tail = parts

        if head == "__system__" and not tail:

                return await self.__system_info__()
            
            # 系统方法不经过中间件
            return await handler(json_rpc_request)
        
        # ... 现有代码 ...
```

### 3. 优化 `AppDoc.docs_json()`

将核心逻辑提取为独立方法，供系统方法调用：

```python
class AppDoc:
    def get_method_tree(self: "RPCServer") -> dict:
        """获取服务器方法树
        
        Returns:
            包含方法树的字典
        """
        # 提取现有 docs_json() 的核心逻辑
        # ...
    
    def docs_json(self: "RPCServer") -> dict:
        """生成当前服务的文档描述数据"""
        tree = self.get_method_tree()
        return {
            "server_name": self.server_name,
            "version": self.version,
            "label": getattr(self, "label", ""),
            "methods": tree["methods"],
            "middlewares": tree["middlewares"],
            "routers": tree["routers"],
        }
```

### 4. 在 RPCClient 中添加便捷方法

为客户端添加获取方法树的便捷方法：

```python
class RPCClient:
    async def get_server_methods(self) -> dict:
        """获取服务器方法树
        
        Returns:
            服务器方法树字典
        """
        future = await self.send("__system__")
        response = await future
        return response.result
```

## 注意事项

1. **命名空间隔离**：系统方法使用特殊命名（如 `__system__`）避免与用户方法冲突
2. **中间件处理**：系统方法是否经过中间件需要明确（建议不经过，保证系统功能可用性）
3. **向后兼容**：确保修改不影响现有功能
4. **文档更新**：更新 README.md 和 AGENTS.md 说明新功能
